## The Problem: Modeling Joint Distributions

An image can be viewed as a collection of random variables, one per pixel. More generally, any observation can be represented as a vector (X₁, X₂, …, Xₙ).

Generative modeling asks a single question: _How do we model the joint distribution p(X₁, X₂, …, Xₙ)?_

If we can model this distribution well, we can sample new data, condition on partial observations or prompts, and infer hidden structure from raw inputs.

For simplicity, we assume binary variables in this chapter. (The same ideas extend to discrete or continuous settings.)

A direct approach is to represent the joint distribution explicitly as a **lookup table**: for every possible configuration of (X₁, …, Xₙ), we store its probability.

If the variables are binary, there are 2ⁿ possible configurations. Since probabilities must sum to one, the number of independent parameters is 2ⁿ − 1. Thus the parameter complexity grows exponentially with n. For high-dimensional data such as images, this representation is computationally impossible.

The central challenge of generative modeling is therefore: _How can we represent the joint distribution without exponential cost?_

---

### The Chain Rule

A natural first step is to rewrite the joint distribution using the **probability chain rule**:

p(X₁, …, Xₙ) = ∏ᵢ p(Xᵢ | X₁, …, Xᵢ₋₁)

This decomposition is always exact — no assumptions are introduced.

However, decomposition is not compression.

If each conditional distribution is represented as a lookup table, then for binary variables the total number of parameters becomes 1 + 2 + 4 + … + 2ⁿ⁻¹ = 2ⁿ − 1, which is still exponential in n.

For example, a modest 32×32 binary image would require 2¹⁰²⁴ entries — clearly infeasible.

The chain rule changes the form of the problem but not its complexity. To make progress, we must introduce additional structure.

---

## Structural Assumptions

The exponential cost arises because each variable is conditioned on _all_ previous variables. In many real systems, however, only a small subset of those dependencies may actually matter.

This motivates a first strategy: _Reduce complexity by restricting which variables each conditional depends on._

### Bayesian Networks

**Definition: Bayesian Network**

A **Bayesian network** is a directed graphical model in which the joint distribution factorizes as:

p(X₁, …, Xₙ) = ∏ᵢ p(Xᵢ | Parents(Xᵢ))

Each variable depends only on a selected subset of variables called its **parents**.

If each variable has at most k parents and variables are binary, each conditional table has 2ᵏ entries. The total parameter count becomes O(n · 2ᵏ), which is linear in n when k is small.

Structural assumptions reduce complexity by limiting dependencies.

### The Markov Assumption

We can push this idea to its extreme by assuming each variable depends only on the immediately preceding one.

**Definition: First-Order Markov Assumption**

The **first-order Markov assumption** states:

Xᵢ₊₁ ⊥ {X₁, …, Xᵢ₋₁} | Xᵢ

The resulting joint distribution becomes:

p(X₁:ₙ) = p(X₁) ∏ᵢ₌₁ⁿ⁻¹ p(Xᵢ₊₁ | Xᵢ)

Here each variable has only one parent, so only local transition probabilities are required.

The Markov assumption is a modeling choice: we intentionally restrict dependencies to obtain tractability.

Both Bayesian networks and Markov chains follow the same philosophy: reduce complexity by narrowing _what we condition on_.

### Why Structure Alone Is Not Enough

Structural assumptions help, but they do not fully solve the problem.

Even when dependencies are sparse, lookup tables scale exponentially with the size of the conditioning context. If the parent set becomes large, tabular representations again become intractable.

This suggests a second, fundamentally different strategy.

---

## Parametric Modeling of Conditional Distributions

Instead of reducing dependencies, we can change how conditionals are represented.

Lookup tables memorize a probability for every possible configuration. But we can instead _compute_ probabilities using a parameterized function.

**Definition: Parametric Conditional Model**

A **parametric conditional model** represents conditional probabilities using a parameterized function:

p(Xᵢ | X<ᵢ) = f_θᵢ(X<ᵢ)

where the function outputs a valid probability distribution. The parameters may be independent for each conditional or shared across conditionals.

The key difference is conceptual: lookup tables memorize each context independently, while functions share parameters and generalize across contexts.

Complexity is now controlled by the function class rather than by the number of possible inputs. This allows us to keep the full chain-rule factorization while avoiding exponential storage.

Generative models become tractable through two complementary design dimensions.

**1. Structural assumptions** simplify the dependency structure by restricting which variables each conditional distribution depends on:

ApproachDependencyParametersRaw chain ruleAll previous variablesO(2ⁿ)Bayesian networkSmall parent set (k parents)O(n · 2ᵏ)Markov chainPrevious variable onlyO(n)

**2. Parametric modeling** simplifies representation by replacing explicit probability tables with parameterized functions. Tabular models store probabilities independently for each context; parametric models compute probabilities using shared parameters.

Structural assumptions control _which dependencies are modeled_, while parametric modeling controls _how conditional distributions are represented_.

These two dimensions are complementary rather than mutually exclusive. Modern generative models often combine both: structural assumptions define the factorization of the joint distribution, while parametric function approximation makes high-dimensional conditionals tractable.

## Autoregressive Models

**Definition: Autoregressive Model**

An **autoregressive model** applies the chain rule factorization exactly:

p(x₁, …, xₙ) = ∏ᵢ₌₁ⁿ p(xᵢ | x₁, …, xᵢ₋₁)

but represents each conditional with a **learned function** f_θᵢ rather than a lookup table. No independence assumptions are introduced — the full dependency structure of the chain rule is preserved.

Sampling proceeds sequentially:

1. Sample x₁ ~ p(x₁)
    
2. Sample x₂ ~ f_θ₂(x₁)
    
3. Sample x₃ ~ f_θ₃(x₁, x₂)
    
4. Continue until xₙ
    

By choosing a specific function class, we implicitly restrict which distributions the model can represent. But this is a far more flexible restriction than hard independence assumptions — and it scales gracefully as the function class becomes more expressive.

**Example:** Consider a sequence of binary variables x₁, x₂, x₃ with learned conditionals:

- p(x₁ = 1) = 0.6
    
- p(x₂ = 1 | x₁) = f_θ₂(x₁)
    
- p(x₃ = 1 | x₁, x₂) = f_θ₃(x₁, x₂)
    

To sample a sequence: draw x₁ with probability 0.6; feed x₁ into f_θ₂ to obtain p(x₂ = 1 | x₁), then sample x₂; feed (x₁, x₂) into f_θ₃ to obtain p(x₃ = 1 | x₁, x₂), then sample x₃. At no point do we enumerate all 2³ = 8 configurations. The function does the work.

---

## Fixed-Length Autoregressive Models

We said that instead of storing p(xᵢ | x₁, …, xᵢ₋₁) as a lookup table, we could _compute_ it using a learned function f_θ. But we never said what f_θᵢ actually looks like.

This section builds up the answer — starting from the simplest possible parameterization and progressively replacing its weaknesses with better ideas. Throughout, we use MNIST as our running example: 28×28 = 784 binary pixels in raster scan order, x₁, …, x₇₈₄ ∈ {0, 1}.

### FVSBN

A common and conceptually simple choice is to model each conditional distribution using logistic regression. In this setting, the conditional probability is parameterized as:

p_θ(xᵢ = 1 | x<ᵢ) = σ(Wᵢ x<ᵢ + bᵢ)

where σ(·) denotes the logistic sigmoid function, so each variable is predicted using a sigmoid applied to a linear function of the previous variables.

The name **Fully Visible Sigmoid Belief Network (FVSBN)** reflects three properties: the model defines probability directly over the observed variables (fully visible), each conditional distribution is parameterized with a sigmoid (logistic regression), and the joint distribution is factorized as a directed probabilistic model — historically called a belief network.

For a conditional depending on m binary variables, a lookup table requires 2ᵐ − 1 independent parameters. If the conditional is instead represented by a parameterized function, such as a logistic function, the number of parameters grows only linearly with m. For example, a linear logistic parameterization requires m+1 parameters. The reduction in complexity comes from restricting the space of possible conditional distributions to those expressible by the chosen function.

**Definition: Fully Visible Sigmoid Belief Network (FVSBN)**

A **Fully Visible Sigmoid Belief Network** parameterizes each conditional as:

x̂ᵢ = p(xᵢ = 1 | x<ᵢ; αⁱ) = σ(αⁱ₀ + ∑ⱼ₌₁ⁱ⁻¹ αⁱⱼ xⱼ)

where σ(z) = 1/(1 + e⁻ᶻ) and αⁱ are learned parameters for position i. The joint distribution is:

p(x₁, …, xₙ) = ∏ᵢ₌₁ⁿ p(xᵢ | x<ᵢ)

**Parameters:** 1 + 2 + … + n = O(n²/2).

**Example: Evaluating and Sampling from FVSBN**

Collect the parameters into a lower-triangular matrix A where row i contains (αⁱ₀, αⁱ₁, …, αⁱᵢ₋₁, 0, …), and let x̃ = (1, x₁, x₂, x₃)ᵀ. Then the vector of conditional probabilities can be written compactly as:

x̂ = σ(A x̃)

where the sigmoid function is applied element-wise.

To evaluate p(X₁=0, X₂=1, X₃=1, X₄=0): compute all x̂ᵢ using the matrix form above, then multiply (1−x̂₁) × x̂₂ × x̂₃ × (1−x̂₄).

To sample: sample x₁ ~ Bernoulli(x̂₁), then sample x₂ ~ Bernoulli(x̂₂(x₁)), and so on.

**The asymmetry of autoregressive models:** Autoregressive models separate evaluation and generation in an important way. Given a known datapoint, the likelihood p(x) can often be computed efficiently, since the conditional terms p(xᵢ | x<ᵢ) are evaluated with inputs that are already known. In contrast, sampling is inherently sequential: each variable must be generated before later conditionals can be evaluated. This difference between efficient likelihood evaluation and sequential generation is a defining property of autoregressive modeling.

FVSBN is valid but limited, since the function f_θ is a logistic regression and therefore models only linear dependencies. To obtain richer, more expressive conditionals, we next replace f_θ with neural networks.

---

### NADE: Neural Autoregressive Distribution Estimators

NADE strengthens autoregressive modeling by replacing the linear conditional predictor of FVSBN with a one-hidden-layer neural network. For each position i, a hidden representation is computed and then mapped to a conditional probability:

hᵢ = σ(Aⁱ x<ᵢ + cⁱ), x̂ᵢ = σ(αⁱ hᵢ + bᵢ)

The hidden layer allows the model to capture nonlinear interactions among observed variables, something a purely linear predictor cannot do.

**The parameter-scaling problem.** Assigning a separate weight matrix Aⁱ ∈ ℝ^(d×(i−1)) to every conditional is costly. Summing over all positions gives:

∑ᵢ₌₁ⁿ d(i−1) = d · n(n−1)/2 = O(n²d)

parameters for the input-to-hidden mappings alone. Beyond the raw count, this design is statistically wasteful: the inputs to adjacent conditionals differ by only a single observation, yet each position learns entirely independent features. There is no mechanism for the model to transfer what it learns about earlier variables to later ones.

**Weight sharing as the key idea.** NADE resolves this by tying the input-to-hidden weights across all positions. A single shared matrix W ∈ ℝ^(d×n) replaces the family {Aⁱ}, and the i-th conditional uses only the first i−1 columns:

hᵢ = σ(W_(:,<i) x<ᵢ + c), x̂ᵢ = p(xᵢ = 1 | x<ᵢ) = σ(αⁱ hᵢ + bᵢ)

Because the same column W_(:,j) appears in every conditional that conditions on xⱼ, the model is forced to learn features that are useful across the entire sequence, not just at a single position. The input-to-hidden parameter count drops to dn = O(nd), and the shared structure enables an efficient left-to-right recurrence: the pre-activation at position i+1 is obtained by augmenting the pre-activation at position i with a single rank-one update, rather than recomputing from scratch.

**Definition: Neural Autoregressive Distribution Estimator (NADE)**

NADE learns a single shared weight matrix W ∈ ℝ^(d×n) so that every conditional reuses the same input-to-hidden mapping:

hᵢ = σ(W_(:,<i) x<ᵢ + cᵢ), x̂ᵢ = p(xᵢ = 1 | x<ᵢ) = σ(αⁱ hᵢ + bᵢ)

Columns of W are shared across positions, so hidden pre-activations can be updated incrementally from left to right. Compared with unshared FVSBN-style networks, NADE reduces input-to-hidden parameter count from O(n²d) to O(nd) while retaining nonlinear expressiveness.

**Extensions.** The NADE framework extends naturally beyond binary variables by modifying the output distribution while retaining the same autoregressive hidden representation.

_Discrete variables._ For non-binary discrete random variables xᵢ ∈ {1, …, K} (e.g., pixel intensities ranging from 0 to 255), the output x̂ᵢ parameterizes a categorical distribution rather than a Bernoulli:

p(xᵢ | x<ᵢ) = Cat(pᵢ¹, …, pᵢᴷ), x̂ᵢ = (pᵢ¹, …, pᵢᴷ) = softmax(Aᵢ hᵢ + bᵢ)

The softmax function generalizes the sigmoid by transforming a vector of real-valued scores into a valid probability distribution over K outcomes, ensuring non-negativity and unit normalization.

_Continuous variables (RNADE)._ For real-valued variables xᵢ ∈ ℝ (e.g., speech signals), each conditional can be modeled using a continuous distribution. A common choice is a mixture of Gaussians, giving rise to the Real-valued NADE (RNADE):

p(xᵢ | x<ᵢ) = ∑ⱼ₌₁ᴷ πᵢʲ · 𝒩(xᵢ; μᵢʲ, σᵢʲ)

where πᵢʲ are mixture weights and (μᵢʲ, σᵢʲ) are the mean and standard deviation of the j-th Gaussian component. These parameters are predicted from the hidden state. In practice, the standard deviations are typically parameterized using an exponential transform to ensure σᵢʲ > 0.

**Remark: Hidden Units as Deterministic Features.** Although hᵢ is commonly called a _hidden layer_, it is more precise to regard it as a **deterministic feature vector** derived from the observations. Given x<ᵢ, the value of hᵢ is fixed: hᵢ = σ(W_(:,<i) x<ᵢ + cᵢ). NADE therefore contains no latent random variables. The hᵢ serve purely as nonlinear summaries of the observed context, used to parameterize each conditional distribution.

### Autoencoders: A Necessary Digression

Before MADE, we need to understand autoencoders — and why they cannot be used as generative models without modification.

**Definition: Autoencoder**

An **autoencoder** maps input x through an encoder f_enc to a representation z, then through a decoder f_dec to a reconstruction x̂:

z = f_enc(x), x̂ = f_dec(z), ℒ = ∑_{x∈𝒟} ∑ᵢ [−xᵢ log x̂ᵢ − (1−xᵢ) log(1−x̂ᵢ)]

The bottleneck at z forces the network to learn a compressed representation. But a vanilla autoencoder is **not** a generative model: it does not define a distribution over x we can sample from.

On the surface, FVSBN and NADE already look like autoencoders: they take x as input and produce x̂ with the same cross-entropy loss. The problem is that a standard autoencoder lets output x̂ᵢ see input xᵢ directly — it can simply copy it, achieving zero reconstruction error while learning nothing useful.

> **The cheating problem:** Output x̂ᵢ must depend _only_ on x₁, …, xᵢ₋₁ to be useful as a generative model. At generation time we do not have xᵢ to copy — that is what we are trying to produce. A standard autoencoder does not enforce this constraint.

---

### MADE: Masked Autoencoder for Distribution Estimation

An ordinary feed-forward autoencoder can be transformed into an autoregressive density estimator by masking its weight matrices. The goal is to preserve the autoregressive factorization:

p(x) = ∏ᵢ₌₁ⁿ p(xᵢ | x<ᵢ)

while computing all conditionals in a **single forward pass**.

**Challenge.** A standard autoencoder is fully connected, meaning each output x̂ᵢ depends on all inputs. This violates the autoregressive constraint, which requires x̂ᵢ to depend only on variables preceding xᵢ in a chosen ordering.

**Key idea: masking as a structural constraint.** MADE enforces a directed acyclic graph (DAG) inside the autoencoder by introducing binary masks on the weight matrices. These masks remove invalid connections so that each output obeys autoregressive dependencies.

Given an ordering (or permutation) π of the variables, the joint distribution factorizes as:

p(x) = ∏ᵢ₌₁ⁿ p(x_πᵢ | x_π<ᵢ)

For example, if π = (x₂, x₃, x₁), then:

p(x₁, x₂, x₃) = p(x₂) · p(x₃ | x₂) · p(x₁ | x₂, x₃)

The output unit producing x̂₂ is therefore not allowed to depend on any input, while the unit producing p(x₃ | x₂) may depend only on x₂, and so forth.

**Hidden-unit degrees.** Instead of manually designing masks, MADE assigns each hidden unit k an integer degree m(k) ∈ {1, …, n−1}. The degree specifies the largest input index that the hidden unit is allowed to depend on. Intuitively, it determines how far into the input ordering the unit can “see.”

**Mask construction.** Masks are constructed using these degrees:

- Input → hidden: a hidden unit with degree m(k) connects only to inputs with index j ≤ m(k).
    
- Hidden → hidden: connections satisfy m(prev) ≤ m(next), ensuring autoregressive consistency across layers.
    
- Hidden → output: output x̂ᵢ receives connections only from hidden units with degree m < i.
    

These constraints guarantee that x̂ᵢ depends only on x<ᵢ, so the network represents a valid autoregressive model.

**Masked forward pass.** Once the degrees are assigned and the masks are constructed, the autoregressive constraints are baked directly into the network weights via the **Hadamard product** (element-wise multiplication, denoted ⊙). For a single hidden layer, the masked forward pass is:

h(x) = g(b + (W ⊙ Mᵂ) x)

x̂ = σ(c + (V ⊙ Mᵛ) h(x))

where g is a nonlinear activation function, σ denotes the sigmoid, b and c are bias vectors, and W, V are the weight matrices for the input-to-hidden and hidden-to-output layers respectively. The binary matrices Mᵂ and Mᵛ are the masks derived from the degree assignments: an entry Mᵂ_{k,j} = 1 if and only if m(k) ≥ j, and an entry Mᵛ_{i,k} = 1 if and only if m(k) < i. By zeroing out the corresponding entries in the weight matrices, the Hadamard product enforces the connectivity constraints structurally, so that no gradient or information can flow through a forbidden path. The entire responsibility of satisfying the autoregressive property therefore rests with Mᵂ and Mᵛ, and the remaining parameters W, V, b, c are free to be optimized by standard backpropagation without any additional constraints.

**Computational structure.** NADE and MADE both exploit shared computations, but differ in how the autoregressive constraint is enforced. NADE computes conditionals through a left-to-right recurrence that incrementally updates hidden pre-activations across positions. MADE instead encodes the same autoregressive dependencies directly in the network connectivity via masks, allowing all outputs to be produced by a single masked forward pass without an explicit sequential recurrence.

**Order-agnostic training.** Autoregressive models usually assume a fixed ordering of variables (e.g., raster scan for images). This introduces an artificial bias, since variables appearing early in the order cannot depend on later ones.

MADE removes this limitation by randomly changing the variable ordering during training and regenerating the corresponding masks. Each ordering defines a different valid autoregressive factorization of the same joint distribution.

Training across many orderings reduces reliance on any single ordering and encourages the model to learn more robust dependencies. This can be viewed as implicitly training an ensemble of autoregressive models while sharing the same parameters.

An additional benefit is flexibility: by choosing an appropriate ordering, inference tasks such as imputing missing values become easier. At sampling time, however, a single ordering is chosen, and variables are generated sequentially according to that order.

**Definition: Masked Autoencoder for Distribution Estimation (MADE)**

MADE is an autoregressive model obtained by applying binary masks to the weight matrices of an autoencoder. Hidden units are assigned ordering degrees that determine allowed connections, ensuring that each output x̂ᵢ depends only on preceding variables. This allows all conditional distributions to be computed in a single forward pass while preserving autoregressive structure.

**Example: MADE with ordering x₂ → x₃ → x₁**

Consider a binary vector x = (x₁, x₂, x₃) with the autoregressive ordering x₂ → x₃ → x₁. The joint distribution becomes:

p(x) = p(x₂) · p(x₃ | x₂) · p(x₁ | x₂, x₃)

_Training._ Given a training example (x₁, x₂, x₃), MADE receives the full binary vector as input. Due to masking, the outputs represent the conditionals: x̂₂ = p(x₂), x̂₃ = p(x₃ | x₂), x̂₁ = p(x₁ | x₂, x₃). Each output is a Bernoulli probability rather than a reconstructed binary value. Parameters are updated using stochastic gradient descent by minimizing the negative log-likelihood (binary cross-entropy).

_Sampling._ After training, sampling follows the chosen ordering, requiring one forward pass per variable.

- **Forward Pass 1.** Input = [0, 0, 0]. MADE runs forward. The output x̂₂ has no dependencies. Sample x₂ ~ Bernoulli(p(x₂)), e.g. x₂ = 1.
    
- **Forward Pass 2.** Input = [0, 1, 0]. MADE runs forward. The masks allow x̂₃ to attend to x₂. Sample x₃ ~ Bernoulli(p(x₃ | x₂ = 1)), e.g. x₃ = 0.
    
- **Forward Pass 3.** Input = [0, 1, 0]. MADE runs forward a final time. The masks allow x̂₁ to attend to both x₂ and x₃. Sample x₁ ~ Bernoulli(p(x₁ | x₂ = 1, x₃ = 0)), e.g. x₁ = 1.
    

The final sampled tuple is (x₁, x₂, x₃) = (1, 1, 0). Although MADE computes all conditionals in one forward pass during training, generation requires D forward passes (one per variable) because each newly sampled variable changes the conditioning context. Only the output corresponding to the next variable in the ordering is read at each step; the remaining outputs are discarded.

### PixelRNN and PixelCNN: Spatial Structure in Images

FVSBN, NADE, and MADE treat an image as a flat sequence, ignoring two-dimensional spatial structure. Real images also have pixel values in {0, …, 255} across three colour channels. For each pixel xₜ, the channels are themselves factorized:

p(xₜ | x<ₜ) = p(xₜʳᵉᵈ | x<ₜ) · p(xₜᵍʳᵉᵉⁿ | x<ₜ, xₜʳᵉᵈ) · p(xₜᵇˡᵘᵉ | x<ₜ, xₜʳᵉᵈ, xₜᵍʳᵉᵉⁿ)

Each channel conditional is a categorical distribution over 256 values.

**Definition: PixelRNN**

**PixelRNN** processes pixels in raster scan order using LSTM variants with masking to preserve the autoregressive ordering. Each pixel’s hidden state hₜ is a _deterministic_ function of all previously observed pixels. It captures long-range spatial dependencies via the recurrent hidden state, gives exact tractable likelihood, but sequential evaluation cannot be parallelized, making it very slow to train. All feature maps are deterministic functions of observed pixels — there are no stochastic latent variables.

**Definition: PixelCNN**

**PixelCNN** replaces recurrence with masked convolutions. Each convolutional filter is zeroed out for all pixels that follow the current position in raster scan order. Stacking masked convolutional layers grows the receptive field while keeping computation parallelizable. Likelihood evaluation is fully parallelizable during training, though generation remains sequential. It achieves similar sample quality to PixelRNN, significantly faster to train.

PixelCNN’s parallelizable training led directly to **PixelDefend** — a defence against adversarial examples that trains a PixelCNN on clean data and flags inputs with anomalously low p(x).

---

### WaveNet: Dilated Convolutions for Audio

The masked convolution idea extends to raw audio. Audio sampled at 16kHz has 16,000 timesteps per second, with dependencies spanning thousands of steps — far beyond a standard convolutional filter.

**Definition: Dilated Convolutions (WaveNet)**

A **dilated convolution** with dilation d skips d−1 samples between each filter tap. Stacking layers with exponentially increasing dilations d = 1, 2, 4, …, 2^(L−1) gives an exponentially growing receptive field:

receptive field = 2^L

Fully parallelizable during training. Generation remains sequential. All computations are deterministic functions of observed audio samples.

### The Fixed-Length Ceiling

All models in this section assume sequence length is fixed at design time. The mask is built for n variables. The architecture has n outputs.

This works for images — 28×28 images always have 784 pixels. It fails completely for language: sentences have different lengths, and a model needs to handle sequences of length 3 and sequences of length 3000 with the same parameters.

This is the boundary between the two sections. The next section introduces the hidden state h as a compressed, dynamically updated context that frees the model from knowing n in advance.

## Variable-Length Autoregressive Models

### Lifting the Fixed-Length Assumption

Every model we studied — FVSBN, NADE, MADE, PixelRNN, PixelCNN, WaveNet — assumed the sequence length n was fixed at design time. The mask was built for exactly n variables. The architecture had exactly n outputs.

This assumption is natural for images. But it is fatal for language, audio transcription, code, or any domain where sequences have different lengths. A model needs to handle a sequence of length 3 and a sequence of length 3000 using exactly the same parameters.

The key insight is to stop asking the architecture to process all previous tokens directly, and instead maintain a **hidden state** hₜ — a fixed-size, dynamically updated compression of everything seen so far. To predict the next token, only hₜ is needed, not the full history. The same parameters work for any sequence length.

**Bridging NADE to RNN.** NADE already contains the core idea that later leads to recurrent models. Instead of recomputing a new network for every conditional distribution, it incrementally updates a running quantity:

aₜ = aₜ₋₁ + W_(:,t−1) xₜ₋₁

which accumulates information about previously observed variables. Each conditional distribution is produced from this accumulated pre-activation. The accumulated vector therefore acts as a primitive form of memory.

However, the accumulation rule in NADE is limited in two ways. First, the update is purely additive and linear: the model learns _what_ to add at each step via the weight columns W_(:,t−1), but not _how_ the accumulated state itself should evolve in response to its current value. Second, the weight columns are position-specific, so the update rule is not shared across steps.

Recurrent neural networks address both limitations through one conceptual change: the state update becomes a learned function shared across all positions:

hₜ = f(hₜ₋₁, xₜ)

where the same function f is applied at every step. The hidden state hₜ is a fixed-dimensional representation summarizing everything observed so far.

In this view, RNNs can be understood as replacing NADE’s handcrafted, position-dependent accumulation rule with a learnable, nonlinear, position-independent update that defines how information is carried forward.

---

### RNNs: The Hidden State as Context

Recurrent Neural Networks (RNNs) were introduced to overcome a core limitation of feed-forward models: the inability to naturally represent sequential structure.

Many forms of data are inherently ordered. Language unfolds token by token, audio evolves continuously over time, and even images may be interpreted as ordered sequences of pixels or patches. In such settings, the prediction at time t should depend not only on the current input xₜ but also on the history x<ₜ.

A standard neural network treats inputs independently and therefore has no mechanism for remembering past observations. RNNs address this by introducing a _hidden state_, a vector that is updated at each time step and acts as a compressed representation of everything seen so far.

**Definition: Recurrent Neural Network (RNN)**

A **Recurrent Neural Network** processes a sequence one element at a time, updating a hidden state hₜ according to:

hₜ = φ₁(W_hh hₜ₋₁ + W_xh xₜ + b_h)

and producing an output:

ŷₜ = φ₂(W_hy hₜ + b_y)

which is typically interpreted as the parameterization of a conditional distribution p(yₜ | hₜ). The parameters W_hh, W_xh, W_hy and biases b_h, b_y are shared across all time steps, allowing sequences of arbitrary length to be processed with a constant number of parameters.

The key idea is simple: rather than explicitly storing all previous inputs, the model learns a function that compresses past information into a fixed-dimensional state. This compact representation enables flexible sequence modeling — but it also introduces a fundamental optimization challenge.

---

## Gradient Flow Through Time

Training an RNN requires learning how information should persist across many steps. This immediately raises a question: _how does learning signal propagate backward through a long sequence?_

Because each hidden state depends on the previous one, gradients must travel through the entire temporal chain. The behavior of this backward flow determines whether the model can learn long-range dependencies.

Assume a sequence of length T, with a loss defined at each step:

L_sequence = ∑ₜ₌₁ᵀ Lₜ

To study gradient propagation, it is sufficient to examine the contribution of a single future loss term L_k (k ≥ t) to an earlier hidden state hₜ. Our goal is therefore to understand how ∂L_k/∂hₜ changes as the temporal distance (k−t) grows.

### Backpropagation Through Time

Since hidden states form a temporal chain hₜ → hₜ₊₁ → … → h_k, the chain rule yields:

∂L_k/∂hₜ = (∂L_k/∂h_k) · ∏ⱼ₌ₜ₊₁ᵏ (∂hⱼ/∂hⱼ₋₁)

Thus, gradients are obtained by repeatedly multiplying Jacobians — one for each recurrent step. The central question becomes: what happens when this multiplication is repeated many times?

### Jacobian of a Recurrent Step

From the update equation hⱼ = φ₁(W_hh hⱼ₋₁ + W_xh xⱼ + b_h), the Jacobian with respect to the previous hidden state is:

∂hⱼ/∂hⱼ₋₁ = Dⱼ W_hh

where Dⱼ = diag(φ₁’(aⱼ)) and aⱼ = W_hh hⱼ₋₁ + W_xh xⱼ + b_h.

The same transformation is therefore applied repeatedly as gradients move backward through time. This repeated reuse of the recurrent dynamics is the source of both the power and the instability of RNN training.

### A Linearized View: Why Gradients Grow or Shrink

To build intuition, suppose the activation derivatives vary slowly across time, so that locally the network behaves approximately linearly, Dⱼ ≈ I. Under this approximation:

∂L_k/∂hₜ ≈ (∂L_k/∂h_k) · W_hh^(k−t)

The problem reduces to repeated powers of the recurrent matrix. If W_hh is diagonalizable with eigenvalues λ₁, …, λₙ, then each component aligned with eigenvector i scales as λᵢ^(k−t).

This immediately reveals the core phenomenon: if |λᵢ| < 1, gradients decay exponentially; if |λᵢ| > 1, gradients grow exponentially. Stable propagation requires eigenvalues whose magnitudes remain close to 1, a condition that is difficult to maintain during training.

### Returning to the Nonlinear Case

The linearized picture provides intuition, but the full network also includes activation derivatives. Taking norms and using submultiplicativity:

‖∂L_k/∂hₜ‖ ≤ ‖∂L_k/∂h_k‖ · ∏ⱼ₌ₜ₊₁ᵏ ‖Dⱼ‖ · ‖W_hh‖

For common activations such as sigmoid or tanh, |φ₁’(x)| ≤ α < 1, which implies ‖Dⱼ‖ ≤ α. Activation derivatives therefore introduce an additional contractive effect, reinforcing the tendency for gradients to shrink over long horizons.

Combined with the recurrent dynamics, this leads to two characteristic pathologies: **vanishing gradients** (early time steps receive almost no learning signal, making long-term dependencies difficult to learn) and **exploding gradients** (gradient norms grow uncontrollably, producing unstable parameter updates).

### Why Exploding Gradients Are Easier to Handle

In practice, exploding gradients are more manageable than vanishing ones. Explosions manifest as sudden large norm spikes, making them detectable during training. A simple and widely used remedy is _gradient clipping_, which rescales gradients whenever they exceed a threshold τ:

g ← g · τ/‖g‖ if ‖g‖ > τ

Vanishing gradients, by contrast, are silent: the learning signal simply disappears, leaving the model unable to adjust parameters that influence distant past states.

> **Key Insight.** Backpropagation through time repeatedly applies the same recurrent transformation. Small deviations from unit magnitude accumulate exponentially: some directions shrink, others grow. As sequence length increases, gradients either vanish or explode, making long-range learning unstable in vanilla RNNs.
> 
> The limitation of vanilla RNNs is not representational power but optimization. A recurrent network can, in principle, model complex sequential dependencies using a fixed set of shared parameters. In practice, however, learning is constrained by unstable gradient flow and by the need to compress an ever-growing history into a finite-dimensional hidden state. These two issues — fragile optimization and memory compression — motivate the architectural developments that follow.

### LSTM and GRU: Fixing the Memory

The core issue is that memory is updated multiplicatively at every step, so gradients must pass through repeated products. Long-range dependencies — the kind that matter most in language — are exactly what vanilla RNNs cannot learn. The LSTM was designed specifically to fix this.

**Definition: Long Short-Term Memory (LSTM)**

The **LSTM** introduces a **cell state** cₜ alongside hₜ, controlled by three learned gates:

- fₜ = σ(W_f [hₜ₋₁, xₜ] + b_f) (forget gate: what to erase from cₜ)
    
- iₜ = σ(W_i [hₜ₋₁, xₜ] + b_i) (input gate: what new info to write)
    
- oₜ = σ(W_o [hₜ₋₁, xₜ] + b_o) (output gate: what to expose as hₜ)
    
- cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ tanh(W_c [hₜ₋₁, xₜ] + b_c)
    
- hₜ = oₜ ⊙ tanh(cₜ)
    

The cell state cₜ is updated by **addition** rather than multiplication. Gradients flowing through cₜ do not shrink exponentially, allowing the LSTM to learn dependencies spanning hundreds of steps. Both hₜ and cₜ are deterministic.

The **Gated Recurrent Unit (GRU)** is a streamlined variant that merges cₜ and hₜ into a single state with two gates instead of three. Computationally cheaper, and in practice often matches LSTM performance.

LSTMs and GRUs dominated sequence modeling from their introduction through the mid-2010s, producing strong results in machine translation, speech recognition, and language modeling. Their fundamental limitation remains: processing position t requires the output of position t−1. Training is inherently sequential and cannot be parallelized across the time dimension.

**Linear accumulation revisited.** The LSTM brings back an idea that first appeared in NADE: a linear additive path for carrying information forward. RNNs replaced linear accumulation with nonlinear state updates to gain expressive power, but this came at a cost: gradients shrink as they pass through many nonlinear transformations, making long-range dependencies hard to learn. The LSTM addresses this by separating two roles that vanilla RNNs conflate. The hidden state handles nonlinear computation at each step. The cell state follows a controlled linear update, gated by what the model has learned to remember or forget, giving gradients a stable path to flow backward across many steps. The key difference from NADE is that the linearity is not fixed: the forget and input gates decide at each step how much of the linear path to use. So the LSTM does not simply recover NADE’s accumulation rule — it places it under learned control. The lesson is that linearity itself is not the problem. Uncontrolled linearity limits what a model can represent. Controlled linearity, combined with nonlinear computation elsewhere, is what makes long-range memory possible.

**Attention.** LSTMs and GRUs solved the vanishing gradient problem but introduced a new one: the entire history of a sequence must be compressed into a fixed-size hidden state. This bottleneck is most visible in encoder-decoder models for machine translation, where the encoder reads the source sentence and produces a single vector that the decoder must use to generate the translation. Performance degrades on long sentences because one vector cannot carry everything the decoder needs.

Attention was introduced as a direct fix. Instead of relying on a single compressed representation, the decoder is allowed to look back at all encoder hidden states and form a weighted sum, placing more weight on positions relevant to the current step. The aggregation is linear — a weighted combination of stored representations — but the weights are learned and input-dependent, giving the model soft control over what to retrieve. This is the same principle that made the LSTM cell state effective: a linear path for information flow, with learned control over what passes through it.

This idea turns out to be more than a patch. If a model can retrieve relevant context from any position directly through attention, the question becomes whether recurrence is needed at all.

**From recurrence to attention.** Recurrence was never the goal; it was simply a mechanism. The real objective has always been to model dependencies between positions in a sequence. Recurrent models do this by passing information forward step by step through a hidden state. Attention provides a different solution: instead of moving information through a chain, each position can directly access other positions. Dependencies are modeled in a single step rather than propagated across many steps.

This change has an important practical consequence. Recurrence is inherently sequential: position t cannot be computed until position t−1 is finished. Attention removes this constraint. All positions can interact in parallel, allowing the entire sequence to be processed simultaneously. As a result, training becomes much faster, especially for long sequences.

One issue remains: causal order. During generation, the prediction at step t must depend only on earlier positions. Recurrence enforces this naturally, but attention requires it to be imposed explicitly. The solution is masking, the same idea used in MADE. Future positions are masked out in the attention matrix so that information cannot flow from the future to the past.

The Transformer follows directly from these ideas: replace recurrence with attention as the main mechanism for modeling dependencies, enforce causal structure through masking, and build the model around parallel, controlled, linear aggregation of information.

### Transformer: Attention Replaces Recurrence

The Transformer replaces recurrence with a different mechanism for modeling context: **self-attention**. Instead of compressing past information into a hidden state that is passed forward step by step, self-attention allows each position to directly interact with all previous positions in a single parallel operation.

**Definition: Scaled Dot-Product Self-Attention**

Each position t produces a **query** qₜ, while every position s produces a **key** kₛ and a **value** vₛ. The output at position t is a weighted sum of values:

Attention(Q, K, V) = softmax(QKᵀ / √d_k) · V

For autoregressive generation, a **causal mask** prevents position t from attending to any future position s > t. This is the attention analogue of MADE’s masking idea: instead of masking network weights, we mask attention connections.

**Definition: GPT (Generative Pre-trained Transformer)**

**GPT** is a decoder-only Transformer that uses causal masked self-attention and is trained autoregressively with the objective:

max_θ ∑ₜ log p_θ(xₜ | x₁, …, xₜ₋₁)

Advantages over RNNs: **parallel training** (all positions are processed at once, removing the sequential bottleneck); **direct long-range access** (any two positions interact through a single attention step); **no compression bottleneck**(attention retrieves relevant information directly rather than storing history in a fixed-size hidden state); **scalability**(performance improves reliably with more data, parameters, and compute).

Generation remains sequential: token t must be produced before token t+1.

The key conceptual shift from RNNs to attention is a move from _compression_ to _retrieval_. An RNN must summarize the entire past into a single hidden state before knowing what future steps will need. Attention postpones this decision: each step directly retrieves whatever past information is most relevant.

**Unifying principles across sequence models.** Across the models in this chapter, two ideas appear repeatedly. The first is **controlled linearity**: a stable linear path for information flow. In LSTMs, the cell state provides a gated linear memory that allows gradients to move across long sequences. In attention, outputs are formed through a learned linear combination of value vectors, where the weights are determined by query–key interactions. The second idea is **masking**, which controls what information is allowed to influence each prediction. MADE uses masking to enforce autoregressive order in feedforward networks, while Transformers use causal masks to enforce the same ordering in attention.

Seen this way, MADE, LSTM, and Transformer offer successive answers to the same questions: how should information flow, and how can that flow remain stable during learning? The Transformer represents the logical end point of this progression: recurrence is removed, information is routed through masked linear attention, and the entire sequence is processed in parallel.
